## Notes on PEISAA Provincials

### Hytek Meet Manager
- The PEISAA Provincial Hytek Meet Manager file should the primary database for
  - events and evt# assignments as it includes and uses all events
  - records as it includes and uses all events
  - Any new event to be added should be added here to guarantee that the evt# is unqiue and consistent across all Zone and regioanls meets
- Each Regional and Zone file will be created by:
  - Copying and renaming OR "Save As"
  - This file will include all events and records
  - Create two sessions (Event/Sessions)
    - Track (which will list the track events held at the meet in order with approx starting times)
    - Field (which will list the field events held at the meet in order with approx starting times)
    - Any events not held at the meet will not appear in the sessions.

- Procedure for carrying records forward
- Issue with some events having inconsistent "Advancement Rules" I set all prelims to have advancement rule based on Place and Time
- Issue with multi-age and records.
- Noticed issue with accented characters. This is caused by using the wrong character set for import from trackie.

### Finishlynx Photofinish
- New folders should be setup for each event (for example)
  - Hytek: One folder with the year and name of event e.g. 2026-PEISAA
    - This folder would hold the meet database and files shared with finishlynx, i.e. lynx.sch, lynx.evt, lynx.ppl and *.lif 
  - Finishlynx: One folder on the local computer for events e.g. 2026-PEISAA
    - The second folder is the Hytek folder listed above
  - The above ensures that data is not overwritten e.g. the files from this meet overwirte the files from last meet.
  - We ran into a situation where a Zone result could no longer be checked because the file had been overwritten. 
- Meet could be run much more efficiently if
  - All athletes wear a competition number. Repeatedly having to ask for athletes names is slow and inaccurate.
    - A clearly visible competition number would speed things up immensely at the callroom, at the start line and at the finish line
  - Distance races (600, 800, 1000, 1200, 1500, 2000 and 3000) would benefit from hip numbers.
    - This would allow the photofinish operator to identify athletes as they cross the finish line and
      not have to wait until all athletes finished and their names reported.  Cost: ???

### Communication
- You need more radios (Callroom, Hytek Operator, Start Line, Photofinish, Finish line, Announcer) at a minimum.
  - With more radios, adjustments to races could be made between the callroom (corral) and the Hytek Operator
  - This allows the Photofinish operator to continue with the current event while the hytek operator prepares the next event
- The start line (Starters Assistant) should report:
  - Event# and Heat# of current event to photofinish
  - all DNS (No Shows) should be reported to the photofinish operator before a race starts.
